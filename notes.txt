## Tổng hợp thông tin dự án — Xổ Số Miền Bắc 5 Phút (pmb5p)

---

### 1. Hệ thống hoạt động thế nào?

- Kết nối WebSocket tới `wss://vip.ee8833.me/ws/` (hoặc `ee8822.me`) để nhận dữ liệu xổ số **thời gian thực**.
- Lọc các gói tin có `id: 45` hoặc `code: "pmb5p"` từ các message loại `lottery_result` và `lottery_info`.
- Tự động lưu vào bộ nhớ: **issue** (mã kỳ), **5 chữ số** (open_numbers_formatted), **tổng**, **Tài/Xỉu**, **Chẵn/Lẻ**.
- Tính toán bằng thuật toán **Xích Markov cấp 1** (Markov Chain) để dự đoán kỳ tiếp theo.

---

### 2. Cách khởi chạy

**Lệnh chạy (từ thư mục `src`):**
```bash
$env:PYTHONPATH=".."; python -m uvicorn main:app --reload --port 8000
```
**Swagger UI:** `http://localhost:8000/docs`

---

### 3. Quy trình vận hành mỗi lần mở máy

**Bước 1 — Cập nhật token (bắt buộc, token hết hạn sau vài giờ):**
- Mở trình duyệt vào `https://vip.ee8833.me` (hoặc `ee8822.me`), đăng nhập.
- F12 → Network → lọc WS → copy giá trị `token=...` trong URL kết nối `/ws/`.
- Gọi `POST /api/config-token` trên Swagger, điền token vào.

**Bước 2 — Nạp lịch sử cũ để tính toán chuẩn:**
- F12 → Network → F5 tải lại trang → lọc `drawResult`.
- Click vào request `drawResult?lottery_id=45...` → tab **Response** → copy toàn bộ JSON.
- Gọi `POST /api/import-history` trên Swagger, dán JSON vào body → Execute.

**Bước 3 — Xem dự đoán:**
- Gọi `GET /api/statistics` để xem kết quả phân tích và dự đoán kỳ tiếp theo.

---

### 4. Cấu trúc dữ liệu quan trọng

| Trường | Ý nghĩa |
|---|---|
| `lottery_id = 45` | Mã game M.bắc nhanh 5 phút |
| `code = "pmb5p"` | Tên code game |
| `open_numbers_formatted` | Mảng 5 chữ số kết quả, ví dụ `["1","6","2","1","6"]` |
| `issue` | Mã kỳ quay, ví dụ `"202607011203"` |
| `is_tai` | Tổng 5 số > 22 = Tài |
| `is_le` | Tổng 5 số lẻ = Lẻ |

---

### 5. Các API Endpoint

| Method | Endpoint | Chức năng |
|---|---|---|
| GET | `/api/history` | Xem lịch sử đã thu thập |
| GET | `/api/statistics` | Xem thống kê & dự đoán kỳ tiếp |
| POST | `/api/config-token` | Cập nhật token WS mới |
| POST | `/api/import-history` | Nạp lịch sử từ Network tab |
| POST | `/api/clear` | Xóa sạch bộ nhớ |

---

### 6. Lưu ý quan trọng

- **Token hết hạn** → hệ thống tự chuyển sang chế độ giả lập (simulation). Log sẽ báo `Running simulation mode`.
- Cần ít nhất **10+ kỳ** mới tính Markov Chain. Khuyến nghị **100+ kỳ** để kết quả dự đoán ý nghĩa.
- Dữ liệu **lưu trong RAM** → khởi động lại server là mất, cần nạp lại.
